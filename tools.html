テストの基本とツール

個別のテストツールと、それらを動かす自動化基盤を整理して理解しましょう。

1. 主要テスト実行ツール

回帰 / E2E

ブラウザ自動操作

  • Playwright: 高速・最新鋭
  • Selenium: 普及率No.1の老舗
  • Cypress: フロントエンド特化
負荷テスト

サーバー耐久性

  • k6: JSで書けるモダンツール
  • JMeter: 多機能な標準ツール
  • Gatling: 高性能な解析向き
TDD / ユニット

コード検証

  • Jest: JS開発の主流
  • JUnit: Javaの金字塔
  • Pytest: Python用の強力ツール

2. 自動化基盤 (CI/CD)

上記のテストツールを「いつ・どこで・自動で実行するか」を管理するオーケストレーターです。

☁️ SaaS / クラウド一体型

  • GitHub Actions: リポジトリ直結で導入が最も手軽。
  • CircleCI: ビルドの爆速さと設定の柔軟性が魅力。
  • GitLab CI: 開発から運用まで1つのツールで完結。

🏗️ セルフホスト / サーバー型

  • Jenkins: 膨大なプラグインを持つ、自動化の巨匠。
  • TeamCity: 高機能かつ美しいUI。Javaやゲーム開発に強み。
  • Bamboo: JiraなどAtlassian製品との連携が最強。
💡

選定のヒント

メンテナンスを減らしたいモダンなプロジェクトは SaaS型。 独自のセキュリティ要件や、複雑なオンプレミス環境を持つ大規模開発は サーバー型 を選ぶのが一般的です。

3. AI駆動型オートメーション

「手順」を教えるのではなく、「目的」を伝えるだけでAIがブラウザを操作する新しい潮流です。

AI Agent

ブラウザ操作代行

  • MultiOn / Skyvern: 自然言語で指示を出すだけで、複雑なWeb操作を完結させる。
  • Operator: OpenAIなどが開発を進める、PC操作全般を代行するエージェント。
Self-Healing

自己修復テスト

  • mabl / Autify: 画面の仕様が変わっても、AIが判断してテストコードを自動修正する。
