テストの基本とツール

実務で使われる主要なツールの特徴を今のうちに押さえておきましょう。

主要テストツールの詳細

回帰 / E2E ブラウザ自動操作ツール

Playwright

Microsoft製の最新鋭ツール。

  • 全ブラウザを1つのAPIで操作可能
  • 動作が非常に高速
Selenium

歴史が長く、最も普及している老舗。

  • 多くの言語（Java, Python等）に対応
  • 学習リソースが豊富
Cypress

フロントエンド特化の高速ツール。

  • ブラウザ内で直接実行される
  • デバッグ体験が非常に優れている

負荷テスト サーバー耐久性検証ツール

Apache JMeter

最も標準的な負荷テストツール。

  • GUIで操作でき、多機能
  • DBやFTPなど多様なテストが可能
k6

エンジニアに人気のモダンなツール。

  • JavaScriptでテストコードが書ける
  • 非常に軽量で自動化しやすい
Gatling

高パフォーマンスな解析ツール。

  • 一度に数万の接続をシミュレート可能
  • 美しいレポート出力機能

TDD / ユニット コード検証フレームワーク

Jest

JS/TS開発のデファクトスタンダード。

  • 設定が簡単で即使える
  • React等の開発で標準的に採用
JUnit

Javaテストの元祖・金字塔。

  • 堅牢なテストスイートを構築可能
  • テスト自動化の歴史そのもの
Pytest

Python用の強力なフレームワーク。

  • 非常にシンプルにテストが書ける
  • プラグインによる拡張性が高い

CI / CD 自動化プラットフォーム (オーケストレーター)

GitHub Actions

GitHub一体型のモダンな自動化ツール。

  • 設定ファイル(YAML)を置くだけで即開始
  • GitHubリポジトリとの親和性が最強
  • 多くのクラウドサービスと連携可能
Jenkins

自由度の高い老舗の自動化サーバー。

  • プラグインが豊富で何でも自動化できる
  • 自前サーバーで運用するためカスタマイズ自在
  • 大規模・複雑なパイプラインに強い
GitLab CI/CD

DevOpsライフサイクルを1つで完結。

  • 計画からデプロイまでを1つの画面で管理
  • セキュリティスキャン等の機能が強力
  • コンテナ(Docker)連携が非常にスムーズ
